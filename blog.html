Question-1:what are the different ways to select an element in the DOM?

In JavaScript, there are several ways to select elements from the DOM, each serving different purposes. The getElementById() method is used to select a single element by its unique id. If you need to select elements by their class name, you can use getElementsByClassName(), which returns a collection of all matching elements. Similarly, getElementsByTagName() allows you to select elements based on their tag name, such as div or p. For more flexibility, querySelector() can be used to select the first element that matches a given CSS selector, while querySelectorAll() selects all matching elements and returns them as a NodeList. Additionally, the getElementsByName() method selects elements based on their name attribute, which is commonly used in form inputs. These methods are essential for interacting with and manipulating HTML elements dynamically through JavaScript.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

The properties innerHTML, innerText, and textContent are all used in JavaScript to access or modify the content of DOM elements, but they serve different purposes. innerHTML allows us to get or set the HTML content of an element, including any tags inside it. This means if the content contains HTML elements like b or span, they will be processed and rendered as part of the content. On the other hand, innerText deals only with the visible text, taking into account the CSS styles that might hide content (like display: none), and it does not return any HTML tags. It focuses on what we actually see on the screen. Meanwhile, textContent retrieves all the text within an element, including hidden elements, and ignores any HTML tags. It returns everything as plain text and is generally faster than innerText. While innerHTML is useful for inserting or retrieving HTML structure, innerText is good for visible content, and textContent is best when we want all the text regardless of how itâ€™s displayed.

Question-3: What is event delegation in the DOM?

Event delegation is a technique where we add an event listener to a parent element instead of each child. It works because events bubble up through the DOM. This allows us to handle events from multiple child elements efficiently, even if they are added dynamically.

Question-4: What is event bubbling in the DOM?

Event bubbling is a behavior in the DOM where an event starts from the target (child) element and then bubbles up to its parent, then the grandparent, and so on up to the root (). For example, if we click a button inside a div, the click event will first trigger on the button, then on the div, and continue up the DOM tree. This allows parent elements to listen for events happening on their child elements. Event bubbling is the foundation for techniques like event delegation.

Question-5: How do you create, add, and remove elements using JavaScript?

In JavaScript, we can create, add, and remove elements in the DOM using built-in methods. Here's a short explanation of each step:


We use document.createElement() to create a new element.

We can add the new element to the DOM using appendChild() or append().
document.body.appendChild(newDiv);
